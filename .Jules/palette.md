## 2026-06-30 - Modal/Drawer Accessibility & Icon Buttons
**Learning:** Drawers and modals often implement custom mask/overlay logic for clicking outside to close, but frequently miss 'Escape' key support. Additionally, icon-only buttons like '×' for closing drawers need `aria-label` attributes to be comprehensible to screen reader users since they lack visual text content.
**Action:** When implementing custom modal or drawer components (like history drawers), always ensure keyboard accessibility by binding 'Escape' to the close action and verify that all icon-only interactive elements possess descriptive `aria-label` attributes.
## 2026-07-08 - Added tooltips and improved localized ARIA attributes
**Learning:** Hardcoded single-language ARIA labels on icon-only links break accessibility when a Vue app relies on dynamic dictionaries. While `aria-label` handles screen readers, adding `title` natively serves sighted users without needing complex tooltip components. Furthermore, toggle buttons (like language switchers) need explicit state (`aria-pressed="true/false"`) rather than relying solely on visual CSS `.active` classes to be fully accessible.
**Action:** When implementing icon buttons and toggles in Vue, always bind both `:aria-label` and `:title` to the `t()` translation function, and explicitly bind `:aria-pressed` based on the active state.
