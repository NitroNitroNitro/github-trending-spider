## 2025-03-09 - [Prevent Log Injection and Memory DoS in AccessLogMiddleware]
**Vulnerability:** The logging middleware blindly trusted user inputs (like `client_ip` and `user_agent`) and wrote them to log files, creating a vulnerability for Log Injection (CRLF injection). Additionally, the in-memory statistics dictionaries `_stats["IP计数"]` and `_stats["接口计数"]` had no size limits, creating a Denial of Service (DoS) vulnerability via memory exhaustion if an attacker spoofed thousands of unique IPs or requested thousands of random paths.
**Learning:** Always sanitize user-provided strings by stripping `\r` and `\n` characters before passing them to the logger. When tracking unbounded user inputs (like IPs or Paths) in memory (e.g. counters or caches), always implement an upper bound on the data structure size to guarantee a maximum memory footprint.
**Prevention:** Sanitize log fields. Implement an upper bound on all in-memory caching or counting mechanisms tracking external inputs.
